This is a "proof of concept" Dashboard screen for your Serendipity blog backend section. Please pass your suggestions and experiences to the serendipity forum.

Its main goal over the time was to get as independent as possible from the old backend view, to be responsive and highly configurable. This is why I forced it to work mainly in full - non-sidebar - screen mode and loading the blogs backend workspace into a modal container. This still has some minor remaining issues (see notes, the unresolved issue list, the changelog, or the todo list) - so you can switch to the old backend whenever you want or need to (please read box notes).

A ruff guide to your new dashboard screen!

  The Dashboard itself has 3 main parts, which are the header and two meta-boxes, one to the left, the other to the right.

  If you hover around with your mouse, you will notice a fine dashed border appear arround a slightly changed background color, which marks the height and width of these meta boxes.

  Inside these meta-boxes, the Dashboard uses draggable content boxes (like general info, pending comments, draft entries, etc).

  If you click an elements boxed title bar, partly every place of an element box, where there is no text and is left blank, the box itself will get marked with a highlighted golden background. Another click will right undo it, if you accidently or by fun clicked the highlight event element box. You can drag this element to the other metabox and drop it there. If you highlight several elements of the same metabox and drag one of them, they will all follow into the dropzone and append to the very end. This drop triggers a cookie and page event to hold you settings until next change. To sort content elements of the same meta-box, just drag and drop them to itself.

  Now, each content-box has the flip-capability to show or hide its content by clicking the small triangle arrow on the right of the boxed title bar. This triggers another event to also be kept and remembered by cookie.

  So, when you return to your start page (the dashboard screen), it will reload the last state of opened or closed content boxes, as well as which was dropped into the meta box zone.

  Some content element boxes, like comments, have another icons to show or hide some additional content features and even some provide a zoom icon, to show truncated content.

  In case you do not want to work with modal containers, push the big blue sliding button to show the sidebar. In this mode, backend links open as they did in old backend mode before.

  And, btw., the state of the sidebar push button will be remembered, as we have seen for the drag&drop boxes and the boxbar-flip event, too.

  In case you need to configure these or other settings by default, you have to click to the dashboard config button , which is in the upper blue header. This opens the plugins configuration where you can finetune the default settings of this plugin.

Have fun!

 

Issue notes!

  When the dashboard page is used in full screen mode (i.e. no sidebar - see push button) and with option set "extended views" to yes in this plugins config, all possible internal backend links get opened in a modal container box like this help screen. This behaviour is still "experimental", since there are some minor issues pending to get closed, which do not behave like in old backend page access.

  This Dashboard revision is a closeup look, mixing new 2.0 development views with a current S9y-1.7 release.

  When using the modal container view by default, the underlying basic dashboard page will not reload its possibly changed values automatically, as of now. If you want to see your changed values, please reload the page by the headers home button, keyboards F5, or your browsers reload button. If you don't want this modal container view in general, you can turn it off in the config options, or leave it in sidebar mode only, which will prevent dashboard boxed buttons & links to open in a modal view container.

Note: If you ever experience weired screen layout issues or discover some other unwanted scripting behaviour on the main dashboard page, do a [Shift] + [Strg/Ctrl] + [r] or [Strg/Ctrl] + [F5] first, to avoid caching or loading issues.

Note: Upon open, the modal window box sometimes (rarely) does not calculate the correct inner-size between container head and foot. Then press the fullview button twice to get the correct size.
